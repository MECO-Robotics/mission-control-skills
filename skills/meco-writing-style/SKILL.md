# MECO Writing Style

Write for students, mentors, and Codex agents who need clear next steps without extra ceremony.

## Workflow

- Be direct, concrete, and concise.
- State what changed, why it matters, and how to verify it.
- Prefer exact file, command, route, and branch names over general descriptions.
- Avoid marketing language, filler, and unexplained acronyms.
- When documenting a workflow, include the commit rule, validation command, and expected failure mode.
