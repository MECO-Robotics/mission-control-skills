# UI Review

Mission Control UI should be compact, operational, and easy to scan during repeated student and mentor use.

## Workflow

- Start from the exact surface named by the task and avoid broadening into adjacent views.
- Reuse existing controls, spacing, icon language, and dark-mode color behavior before inventing new patterns.
- Keep disabled or unavailable views visible when that preserves user orientation.
- Check responsive behavior for text fit, overlap, tap targets, and state changes.
- Validate rendered behavior with the app-specific local or emulator workflow when UI behavior changes.
